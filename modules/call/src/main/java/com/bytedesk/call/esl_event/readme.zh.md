# 流媒体源配置

用来把可持续播放的音频流（常见是网络 URL）接入电话系统。

## 主要用途

作为 MOH/公告音源：例如在线电台、云端音频流。
在 IVR 或其他呼叫流程中作为播放目标（按业务规则引用）。
统一管理流地址、名称、启用状态，避免在 dialplan 里硬编码 URL。
和 recordings / music_on_hold 的区别

recordings：本地或库内静态音频文件。
music_on_hold：等待音乐分类与文件管理（多为本地素材）。
esl_event：外部或实时音频流源（偏“动态来源”）。

## 适合场景

需要频繁更新内容且不想反复上传文件。
想把第三方音频服务接入 PBX 播放链路。

一句话：esl_event 是“可复用的流媒体音源清单”，让通话流程可以直接消费实时/远程音频流。
